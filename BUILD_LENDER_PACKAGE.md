## Feature 1: Lender/Investor Package — Shareable Compliance Report

PermitFlow can't do this. One-click generates a shareable link with a professional compliance report for lenders, investors, and attorneys.

### 1a. Share Page — `src/app/share/[token]/page.tsx`
Public page (no auth) that shows a read-only compliance report for a project.

**URL:** /share/[shareToken]

**Content:**
- MeritLayer header + "Compliance Report" label
- Project name, address, report generated date
- **Compliance Summary Card:** Score %, items complete/total, status badge
- **Upcoming Deadlines:** Next 3 deadlines with dates
- **Permit Workflow:** All permits + current status
- **Compliance Checklist:** All items with status icons (✅ ⏳ ❌)
- **Document Log:** List of uploaded documents
- Footer: "Generated by MeritLayer · meritlayer.ai" + "Get started for free"

This page is read-only, no editing, professional appearance.

### 1b. DB Schema addition in `src/db/schema.ts` (append only):
```typescript
export const projectShares = pgTable("project_shares", {
  id: text("id").primaryKey().$defaultFn(() => crypto.randomUUID()),
  projectId: text("project_id").notNull().references(() => projects.id, { onDelete: "cascade" }),
  shareToken: text("share_token").notNull().unique().$defaultFn(() => crypto.randomUUID()),
  createdBy: text("created_by").notNull(),
  label: text("label"), // e.g. "For First Republic Bank"
  expiresAt: timestamp("expires_at"), // null = never expires
  viewCount: integer("view_count").default(0),
  createdAt: timestamp("created_at").defaultNow(),
});
```

### 1c. Migration — `drizzle/0005_project_shares.sql`
```sql
CREATE TABLE IF NOT EXISTS "project_shares" (
  "id" text PRIMARY KEY NOT NULL,
  "project_id" text NOT NULL REFERENCES "projects"("id") ON DELETE CASCADE,
  "share_token" text NOT NULL UNIQUE,
  "created_by" text NOT NULL,
  "label" text,
  "expires_at" timestamp,
  "view_count" integer DEFAULT 0,
  "created_at" timestamp DEFAULT now()
);
CREATE INDEX IF NOT EXISTS "project_shares_token_idx" ON "project_shares"("share_token");
CREATE INDEX IF NOT EXISTS "project_shares_project_idx" ON "project_shares"("project_id");
```

### 1d. tRPC Router — `src/server/api/routers/shares.ts`
Procedures:
- `create({ projectId, label, expiresInDays? })` → creates share record, returns shareToken and full URL
- `getByProject(projectId)` → list all shares for a project
- `getByToken(token)` → get project data for the share page (public, no auth), increments viewCount
- `revoke(shareId)` → delete the share (owner only)

Add to `src/server/api/root.ts` — import and add `shares: sharesRouter`

### 1e. Share Button in project page
In `src/app/(dashboard)/projects/[id]/page.tsx`, find the page header and ADD a "Share Report" button next to the existing buttons. Opens a small dialog:
- "Create Shareable Link" title

- "Generate Link" button → calls create mutation → shows the link with copy button
- "This link gives read-only access. Anyone with the link can view this report."
- List of existing shares with "Revoke" buttons

## Feature 2: AI Document Chat — "Ask Your Documents"

PermitFlow has zero AI. This is a massive differentiator. User can ask questions about their uploaded documents.

### 2a. Chat Component — `src/components/document-chat.tsx`
A chat interface that lives in a tab on the project page.

UI:
- Chat messages display (user + assistant bubbles)
- Input at bottom: "Ask anything about your project documents..."
- Example prompts as clickable chips: "What's my BPDA deadline?", "What are my parking requirements?", "Summarize the compliance items", "What documents am I missing?"
- Loading state while AI thinks
